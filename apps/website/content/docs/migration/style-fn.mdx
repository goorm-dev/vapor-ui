## 개요

`$style({...})`은 빌드 타임 매크로다. 호출 사이트가 atomic class 문자열 리터럴로 치환되고, 매크로가 생성한 CSS chunk가 자동 import된다. 런타임에는 함수가 실행되지 않는다.

```tsx
import { $style } from '@vapor-ui/core';

const Box = () => (
    <div
        className={$style({
            padding: { default: '$400', sm: '$200' },
            backgroundColor: '$blue-500',
            color: { default: '$white', _hover: '$blue-100' },
        })}
    />
);
```

## 번들러 설정

### Vite

```ts
// vite.config.ts
import vaporStyleMacro from '@vapor-ui/style-macro/unplugin';
export default { plugins: [vaporStyleMacro.vite()] };
```

### Next.js (webpack)

```js
// next.config.mjs
import vaporStyleMacro from '@vapor-ui/style-macro/unplugin';
const vaporPlugin = vaporStyleMacro.webpack();

export default {
    webpack(config) {
        config.plugins.push(vaporPlugin);
        return config;
    },
};
```

> Turbopack은 미지원. `next build --webpack` / `next dev --webpack` 사용.

### Rollup

```js
// rollup.config.mjs
import vaporStyleMacro from '@vapor-ui/style-macro/unplugin';
export default { plugins: [vaporStyleMacro.rollup()] };
```

## Breakpoint custom-media

`sm`/`md`/`lg` 키는 `@media (--vapor-sm/md/lg)`로 컴파일된다. 이걸 실제 미디어 쿼리로 확장하려면 `@vapor-ui/core/postcss` helper를 PostCSS 파이프라인에 추가:

```js
// postcss.config.mjs
export default {
    plugins: {
        '@vapor-ui/core/postcss': {},
        '@tailwindcss/postcss': {},
    },
};
```

override 필요 시 `vaporCustomMedia({ sm, md, lg })` 직접 호출.

기본값:
- `--vapor-sm` → `(max-width: 767px)`
- `--vapor-md` → `(max-width: 1024px)`
- `--vapor-lg` → `(min-width: 1025px)`

## 지원하는 입력

값은 다음 중 하나:
- 문자열 리터럴 (`'flex'`, `'center'`)
- 숫자 (`8`)
- 토큰 (`'$400'`, `'$blue-500'`) — `$<name>` prefix
- entry-level ternary (`isLarge ? '$600' : '$400'`)

condition object 키:
- `default` — base
- `sm` / `md` / `lg` — named breakpoint
- `'@media (...)'` — raw media query
- `_before` / `_after` / `_hover` / `_focus` / `_focusVisible` / `_focusWithin` / `_active` — pseudo

## 제약

- spread (`...base`), computed key (`[k]: ...`), dynamic value (identifier/expression) 모두 build error
- condition 안에서는 ternary 금지 (entry-level만)
- 알 수 없는 token → `unknown-token` / `scope-mismatch` build error + codeframe

## `$css` 와 차이

| | `$css` | `$style` |
|---|---|---|
| 실행 시점 | 런타임 (rainbow-sprinkles) | 빌드 타임 |
| 산출물 | inline style + dynamic className | atomic class string + CSS chunk |
| 런타임 비용 | 있음 | 0 |
| dynamic value | 가능 | ternary만 |
