# @vapor-ui/translation-pipeline

An internal CLI package that translates English API JSON generated by `ts-api-extractor` into Korean documentation JSON. It calls an LLM via LiteLLM and validates translation quality through an MQM (Multidimensional Quality Metrics) gate.

## Flow

```
packages/core components
    ↓ ts-api-extractor
generated/en/*.json
    ↓ translation-pipeline
generated/ko/*.json
.translation-cache.json
.i18n-report.md
```

`apps/website` runs both tools in sequence.

```bash
pnpm --filter website i18n       # extract + translate
pnpm --filter website extract    # extract only
pnpm --filter website translate  # translate only
```

## Requirements

- Node.js >= 20.20.2
- LiteLLM API key and endpoint (see team credentials doc)

## Environment Variables

Must be set before running the CLI. The CLI automatically reads `.env` in the current working directory.

```bash
LITELLM_API_KEY=<api key>
LITELLM_BASE_URL=<litellm endpoint>
```

## CLI Options

```bash
translation-pipeline --input <dir> --output <dir>
```

| Option     | Short | Required | Description                                 |
| ---------- | ----- | -------- | ------------------------------------------- |
| `--input`  | `-i`  | Yes      | Directory containing English JSON files     |
| `--output` | `-o`  | Yes      | Directory to write `ko/`, cache, and report |

## Input and Output

Only `description` fields are translated.

- Component description: `description`
- Prop description: `props[].description`

All other fields (`name`, `type`, `required`, `defaultValue`, etc.) are preserved as-is.

| Output                    | Description                           |
| ------------------------- | ------------------------------------- |
| `ko/*.json`               | JSON with Korean descriptions applied |
| `.translation-cache.json` | Verified translation cache            |
| `.i18n-report.md`         | MQM quality report                    |

## Code Structure

```
src/
├── cli/           # Entry point, argument parsing, file I/O
├── translator/    # Unit collection, cache lookup, batch lifecycle
├── translation/   # LiteLLM calls, initial translation prompt
├── validation/    # MQM evaluation prompt and response validation
├── cache/         # SHA256 key generation, cache load/save
├── report/        # Per-component stats, report rendering
├── defaults.ts    # Default model names
└── types.ts       # Shared type definitions
```
